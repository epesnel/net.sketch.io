# net.sketch.io

TF/Keras model visualizer — paste Python code, get an architecture graph.

## Architecture

Static analysis pipeline, all in JS (no backend):
1. `src/tokenizer.js` — Python tokenizer with indentation tracking (INDENT/DEDENT)
2. `src/parser.js` — Recursive descent + Pratt parser → AST
3. `src/interpreter.js` — Partial evaluator: traces `__init__` + `call()`, builds computation graph with shape inference
4. `src/renderer.js` — SVG graph rendering with zoom/pan

## Build

No build step. For development, `app.html` is a self-contained file with all JS/CSS inlined (generated by a Python script). The source modules in `src/` are the canonical code.

To rebuild `app.html` after editing `src/*.js` or `style.css`:
```bash
python3 build.py  # TODO: create this
```

For now, the rebuild is done inline in the conversation via a Python script that:
1. Reads all `src/*.js` files, strips `export` keywords
2. Concatenates into a bundle
3. Inlines the bundle + CSS + app logic into `app.html`

## Testing

`tests/` contains test HTML pages. Run with a local HTTP server:
```bash
python3 -c "from http.server import HTTPServer, SimpleHTTPRequestHandler; from socketserver import ThreadingMixIn; import os; os.chdir('.'); type('T',(ThreadingMixIn,HTTPServer),{'daemon_threads':True})(('',9090),SimpleHTTPRequestHandler).serve_forever()" &
open http://localhost:9090/tests/test_pipeline.html
```

No Node.js available. Tests run in the browser.

## Known limitations

- Top-level `await` in `<script type="module">` doesn't work reliably — use `async function` wrapper
- ES module `<script type="module" src="...">` fails silently in some cases — `app.html` uses inline `<script>` with bundled code instead
- Python `http.server` is single-threaded — use `ThreadingMixIn` for concurrent module loading
- Static analysis can't resolve dynamic shapes (`tf.shape()`, `tf.cast()`, runtime slicing)
- `for i in range(...)` targets must use `parseExpr(5)` to avoid consuming `in` as comparison operator

## Key bugs fixed

- Tokenizer: blank lines set `atLineStart=true` but `readNext` didn't reset it → spurious INDENT mid-line. Fix: `if (this.atLineStart) continue;` after `processIndentation()`
- Parser: `parseForTarget` consumed `in` as comparison operator. Fix: `parseExpr(5)` (binding power above comparison)
- Parser: DEDENT at module level caused infinite loop in error recovery. Fix: explicit DEDENT skip + forced advance guard
- Parser: tuple unpacking `a, b = x, y` not parsed. Fix: check for `,` before `=` in `parseAssignOrExpr`
